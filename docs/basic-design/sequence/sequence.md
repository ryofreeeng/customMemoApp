# シーケンス図（第1サイクル：メモ一覧・新規作成・移動先選択）

基本設計フェーズ・第1サイクルの3画面を対象とした、処理の流れを示す図。

**登場人物**
- **ユーザー**：アプリを操作する人
- **アプリ**：React Native フロントエンド（画面表示・ユーザー操作の受付）
- **SQLite**：端末内のローカルデータベース

**設計方針メモ**
- 操作メニュー（長押し・スワイプボタン）の表示はフロントエンドのみで完結。DBアクセス不要
- DBを触るのは「実際にアクションを実行するとき」のみ（作成・移動・削除など）
- 具体的なSQL文・エラーハンドリング・ローディング状態の制御は詳細設計で扱う

---

## フロー1: メモ一覧の初期表示

```mermaid
sequenceDiagram
    actor U as ユーザー
    participant A as アプリ
    participant DB as SQLite

    U->>A: アプリを起動
    A->>DB: フォルダ一覧を取得（ルート直下・五十音順）
    A->>DB: メモ一覧を取得（ルート直下・最終更新日降順）
    DB-->>A: フォルダ一覧・メモ一覧
    A-->>U: メモ一覧画面を表示
```

---

## フロー2: フォルダタップ（階層を深く入る）

```mermaid
sequenceDiagram
    actor U as ユーザー
    participant A as アプリ
    participant DB as SQLite

    U->>A: フォルダをタップ
    A->>DB: タップしたフォルダ直下のフォルダ一覧を取得（五十音順）
    A->>DB: タップしたフォルダ直下のメモ一覧を取得（最終更新日降順）
    DB-->>A: フォルダ一覧・メモ一覧
    A-->>U: メモ一覧画面を更新（タップしたフォルダの中身を表示）
```

---

## フロー3: 操作メニュー・スワイプボタンの表示（DBアクセスなし）

長押しメニュー・左スワイプボタンはフロントエンドに定義されたUIで、どのアイテムも同じ選択肢を持つ。表示時にDBを触る必要はない。

```mermaid
sequenceDiagram
    actor U as ユーザー
    participant A as アプリ
    participant DB as SQLite

    Note over A,DB: 表示のみ。DBアクセスは発生しない。

    U->>A: アイテムを長押し
    A-->>U: 操作メニューを表示（移動・削除・名前変更）

    U->>A: アイテムを左スワイプ
    A-->>U: 移動・削除ボタンを表示（アイテム右側）
```

## フロー4: 新規アイテム作成

```mermaid
sequenceDiagram
    actor U as ユーザー
    participant A as アプリ
    participant DB as SQLite

    U->>A: ＋新規作成をタップ
    A-->>U: 種別選択ボトムシートを表示
    U->>A: 「フォルダを作成」または「メモを作成」を選択
    A-->>U: 一覧の先頭に仮名アイテムを追加して編集状態で表示（キーボードが出る）

    alt 名前ありで確定
        U->>A: 名前を入力して確定ボタン or 外タップ
        A->>DB: INSERT（FOLDERまたはNOTEテーブルに新規レコード追加）
        DB-->>A: 成功
        A-->>U: 一覧を更新（フォルダ→五十音順の正式な位置へ移動、メモ→先頭のまま）
    else 名前が空欄で確定 or 外タップ
        U->>A: 空欄のまま確定 or 外タップ
        A-->>U: 仮名アイテムを一覧から除去（DBへの書き込みなし）
    end
```

## フロー5: アイテムの移動

```mermaid
sequenceDiagram
    actor U as ユーザー
    participant A as アプリ
    participant DB as SQLite

    U->>A: 左スワイプ→移動タップ（または長押し→移動）
    A->>DB: フォルダ一覧を取得（移動先ピッカー用・ルートから）
    DB-->>A: フォルダ一覧・メモ一覧
    A-->>U: 移動先選択画面を表示

    loop 移動先フォルダを探して掘り進む（0回以上）
        U->>A: フォルダをタップ
        A->>DB: タップしたフォルダ直下のフォルダ一覧・メモ一覧を取得
        DB-->>A: フォルダ一覧・メモ一覧
        A-->>U: 移動先選択画面を更新（タップしたフォルダの中身）
    end

    alt 移動を確定
        U->>A: 「ここに移動」をタップ
        A->>DB: UPDATE（移動対象の parent_id または folder_id を移動先に変更）
        DB-->>A: 成功
        A->>DB: 元フォルダの一覧を再取得
        DB-->>A: 更新後の一覧
        A-->>U: 移動先選択画面を閉じ、元フォルダの一覧を表示（移動済みアイテムは消えている）
    else キャンセル
        U->>A: キャンセルタップ（右上✕または下部キャンセルボタン）
        A-->>U: 移動先選択画面を閉じ、元フォルダの一覧をそのまま表示
    end
```
