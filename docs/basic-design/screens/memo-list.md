# 画面設計書：メモ一覧（フォルダビュー）

要件定義（`docs/requirements/screens.md`の画面一覧表#1）に対応する画面設計書。レイアウトは[memo-list.excalidraw](./memo-list.excalidraw)を参照。

## 画面の役割
フォルダ階層をたどり、メモを探す起点となる画面。フォルダを開く・作る・名前を変える・移動する・削除する、メモを選んで開くことができる。

## 項目一覧表

| 項目名 | 種別 | データ元 | イベント |
|---|---|---|---|
| ＜戻る | アウトプット（操作トリガー） | なし | タップで1つ上のフォルダ（親フォルダ）の一覧に戻る。ルートフォルダの場合は非表示。画面全体の右スワイプでも同じ動作（5-4章） |
| フォルダ名（タイトル） | アウトプット | `FOLDER.name`（現在開いているフォルダ） | なし（表示のみ）。ルートフォルダの場合は表示しない（空欄） |
| ⚙設定 | アウトプット（操作トリガー） | なし | タップで設定画面に遷移 |
| リスト項目（📁フォルダ） | アウトプット | `FOLDER`テーブルから`parent_id`＝現在のフォルダのレコードを取得 | タップでそのフォルダの一覧に遷移。長押しで操作メニューを表示。左スワイプで移動・削除ボタンを表示。右スワイプでピン留めボタンを表示（拡張フェーズ） |
| リスト項目（📄メモ） | アウトプット | `NOTE`テーブルから`folder_id`＝現在のフォルダのレコードを取得 | タップでメモ詳細画面に遷移。長押しで操作メニューを表示。左スワイプで移動・削除ボタンを表示。右スワイプでピン留めボタンを表示（拡張フェーズ） |
| ＋新規作成 | インプット（操作トリガー） | なし | タップでフォルダ/メモのどちらを作るか選び、現在のフォルダの中に新規作成する |
| 🔍検索 | アウトプット（操作トリガー） | なし | タップで検索画面に遷移 |
| ☑選択 | インプット（操作トリガー） | なし | タップで複数選択モードをON/OFFする（v1ではUI配置のみ・タップ未対応。拡張フェーズで機能を実装） |
| 画面全体 | アウトプット（操作トリガー） | なし | 右スワイプ（画面の左端からのみ開始）で1つ上の親フォルダの一覧に戻る（5-4章）。ルートフォルダの場合は何も起きない |

## 下部バーのレイアウト

画面下部を3等分し、左から順に配置する。

```
┌─────────────┬─────────────┬─────────────┐
│  ＋ 新規作成  │   🔍 検索   │   ☑ 選択    │
└─────────────┴─────────────┴─────────────┘
```

複数選択モードがONの間は、下部バーが以下に切り替わる（拡張フェーズ）：

```
┌─────────────┬─────────────┬─────────────┐
│    移動     │    削除     │   ☑ 選択    │
└─────────────┴─────────────┴─────────────┘
```

☑選択は右端に固定したまま、左2つが移動・削除に変わる。再度タップで選択モードをOFFにできる。

## 操作メニュー（長押しでフォルダ/メモの上に表示）

フォルダ・メモのどちらを長押しした場合も、同じ4つの選択肢を表示する。

| 項目名 | イベント |
|---|---|
| 名前を変更 | タップでその場でテキスト編集できる状態にする（インライン編集）。見えている範囲で編集するため、編集後の表示がイメージしやすい |
| 移動 | タップで[移動先選択画面](./move-destination.md)に遷移する |
| タグを編集 | タップでタグ付け編集UIを表示する（UIの詳細は各画面の詳細設計時に確定） |
| 削除 | タップで即座にゴミ箱へ移動。その後5秒間だけ「削除を取り消す」トーストを表示し、タップで元に戻せる（トーストの位置は後述） |

## 削除取り消しトーストの表示位置

画面右下・下部バーのギリギリ上（バーと重ならない位置）に表示する。左側のメモタイトルと被らないよう、右寄せで配置する。

```
┌────────────────────────────────────┐
│ 📄 今日のタスク                    │
│ 📄 買い物リスト                    │
│ 📄 アイデアメモ                    │
│                    ┌────────────┐  │
│                    │削除を取り消す│ ← ここ
│                    └────────────┘  │
├──────────┬──────────┬──────────────┤
│ ＋新規作成 │ 🔍検索   │ ☑ 選択      │
└──────────┴──────────┴──────────────┘
```

## 左スワイプ操作（アイテム個別）

リスト項目を左にスワイプすると、アイテムが左にスライドして右側にボタンが出現する。

| ボタン | イベント |
|---|---|
| 移動 | タップで[移動先選択画面](./move-destination.md)に遷移する |
| 削除 | タップで即座にゴミ箱へ移動。その後5秒間だけ「削除を取り消す」トーストを表示し、タップで元に戻せる（トーストの位置は前述） |

```
左スワイプ中（「今日のタスク」を左スワイプした状態）:
┌──────────────────────────────────────────┐
│ ＜ 仕事フォルダ                     ⚙   │
├──────────────────────────────────────────┤
│  📁 プロジェクトA                       │
│  📁 プロジェクトB                       │
│  📄 今日のタスク ◁──│  移動  │  削除  │
│  📄 買い物リスト                        │
│  📄 アイデアメモ                        │
│                        ┌────────────┐   │
│                        │削除を取り消す│  │
│                        └────────────┘   │
├──────────────┬──────────┬────────────────┤
│  ＋ 新規作成  │ 🔍 検索  │   ☑ 選択      │
└──────────────┴──────────┴────────────────┘
※トーストは削除ボタンをタップしたときのみ表示
```

## 右スワイプ操作（アイテム個別）【拡張フェーズ】

リスト項目を右にスワイプすると、アイテムが右にスライドして左側にボタンが出現する。

| ボタン | イベント |
|---|---|
| ピン留め | タップでそのフォルダ内の上部に固定表示する |

```
右スワイプ中（「今日のタスク」を右スワイプした状態）【拡張フェーズ】:
┌──────────────────────────────────────────┐
│ ＜ 仕事フォルダ                     ⚙   │
├──────────────────────────────────────────┤
│  📁 プロジェクトA                       │
│  📁 プロジェクトB                       │
│ │ピン留め│  📄 今日のタスク ▷──        │
│  📄 買い物リスト                        │
│  📄 アイデアメモ                        │
│                                          │
├──────────────┬──────────┬────────────────┤
│  ＋ 新規作成  │ 🔍 検索  │   ☑ 選択      │
└──────────────┴──────────┴────────────────┘
```

※画面全体の右スワイプ（親フォルダに戻る）との競合について：画面左端からのスワイプは親フォルダへの戻り操作に使用するため、アイテムへの右スワイプは左端ゾーンを除いたアイテム上からのみ有効とする。

## タグの表示仕様

### 表示位置

各リスト行の上辺の外側（カード上部の固定ギャップ内）に小さな色付きタブとして表示する。ギャップは**タグの有無にかかわらず常に一定の高さ（約14〜16px）を確保**する（条件付きレイアウトを避けるため）。タブの高さは約10px、幅は約28px。

```
[固定ギャップ（14px）] ← ここにタグタブが出る
┌────────────────────────────────────┐
│  📄 今後やりたいこと               │  ← カード本体（45px）
└────────────────────────────────────┘
[固定ギャップ（14px）]
```

フォルダ行・タグなしのメモ行は、ギャップが空白のまま表示される。

### タブの内容・上限

| 条件 | 表示 |
|---|---|
| タグ1〜3件 | タグの色で塗りつぶした小さな長方形タブを横に並べる |
| タグ4件以上 | 上位3件のタブ ＋ 残り件数を`+N`テキストで表示（例：タグが5件なら`+2`） |
| タグなし | ギャップは空白のまま（タブは表示しない） |

### タグの表示は検索補助用途

タグはあくまで検索時の絞り込みが主用途。一覧での色タブ表示は「このメモにタグが付いている」と気づく補助的な情報として機能する。

## メモ一覧の表示順・レイアウト

### 表示順（確定）

同じフォルダの中の表示は、以下のルールで並べる。

| 種別 | 並び順 |
|---|---|
| 📁 フォルダ | 五十音順（昇順）。常に上部に表示 |
| 📄 メモ | 最終更新日が新しい順（降順）。フォルダの下に表示 |

```
例：
📁 会議メモ       ← フォルダ（五十音順）
📁 仕事
📄 今日のタスク   ← メモ（最終更新日順）
📄 アイデアメモ
```

### レイアウト切り替え【拡張フェーズ】

⚙設定メニューから以下の2パターンを切り替えられる。デフォルトは「連続表示」。日付区切りはメモのみに適用（フォルダは常に五十音順）。

| パターン | 概要 |
|---|---|
| 連続表示（デフォルト） | フォルダ（五十音順）→メモ（最終更新日順）を区切りなく表示 |
| 日付区切り表示 | メモを更新日ごとにグループ化し、グループ間に空白を挿入して視覚的に区切る |

## 複数選択モード【拡張フェーズ】

☑選択ボタンをタップするとON（ボタンに色がつく）。この状態でリスト項目をタップすると選択状態になる。

- 1件以上選択中：下部バーが「移動」「削除」に切り替わる（☑選択は右端に残る）
- 選択キャンセル：色のついた☑選択ボタンをもう一度タップ→元の下部バーに戻る

```
複数選択モードON（2件選択済みの状態）【拡張フェーズ】:
┌──────────────────────────────────────────┐
│ ＜ 仕事フォルダ                     ⚙   │
├──────────────────────────────────────────┤
│ ○  📁 プロジェクトA                     │
│ ●  📁 プロジェクトB     ← 選択済み      │
│ ○  📄 今日のタスク                      │
│ ●  📄 買い物リスト      ← 選択済み      │
│ ○  📄 アイデアメモ                      │
│                                          │
├──────────────┬──────────┬────────────────┤
│    移動      │   削除   │ ☑ 選択（ON）  │
└──────────────┴──────────┴────────────────┘
○ = 未選択　● = 選択済み
```
