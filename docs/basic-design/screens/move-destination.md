# 画面設計書：移動先選択

要件定義（`docs/requirements/screens.md`の画面一覧表#10）に対応する画面設計書。レイアウトは[move-destination.excalidraw](./move-destination.excalidraw)を参照。[memo-list.md](./memo-list.md)の長押しメニューから「移動」を選んだときに、元の画面の手前にほぼ全画面で重なって表示される。

奥の画面（元のメモ一覧等）がどう見えるかは、要件定義5-2章「重なりの視認性」（奥の層を完全に隠さず、少し小さく・ずらして・少し暗く表示する）のルールがこの画面にも適用される。重なった状態のイメージは[move-destination-overlay.excalidraw](./move-destination-overlay.excalidraw)を参照。

## 画面の役割
フォルダ・メモの移動先となるフォルダを選ぶための画面。メモ一覧画面とほぼ同じレイアウトを使う。フォルダはタップして移動先として選べるが、**メモは非活性（タップ不可・グレーアウト表示）**で、移動先選択の対象にはならない。メモも表示することで、「このフォルダにはこのメモが入っている」という手がかりになり、フォルダを見分けやすくする。

## 項目一覧表

| 項目名 | 種別 | データ元 | イベント |
|---|---|---|---|
| 「移動先を選択」（タイトル） | アウトプット | なし（固定文字列） | 表示のみ |
| × キャンセル（右上） | インプット（操作トリガー） | なし | タップでこの画面を閉じ、移動操作自体を中止して元の画面に戻る |
| ＜ 戻る（左上） | インプット（操作トリガー） | なし | タップでこの画面の中で1階層浅いフォルダの表示に戻る（移動操作自体は中止しない）。サブフォルダに入っているときのみ表示し、ルートフォルダ表示中は非表示 |
| リスト項目（📁フォルダ） | アウトプット | `FOLDER`テーブルから`parent_id`＝現在表示中のフォルダのレコードを取得（移動対象のフォルダ自身は除外する） | タップでそのフォルダの中に移動（画面内をさらに掘り進む） |
| リスト項目（📄メモ・非活性） | アウトプット | `NOTE`テーブルから`folder_id`＝現在表示中のフォルダのレコードを取得 | 表示のみ（グレーアウト）。タップしても何も起きない |
| 画面全体 | アウトプット（操作トリガー） | なし | 右スワイプで＜戻るボタンと同じ動作（1階層浅いフォルダに戻る）。移動操作自体は中止しない |
| キャンセル（下部・左） | インプット（操作トリガー） | なし | タップでこの画面を閉じ、移動操作自体を中止して元の画面に戻る。×ボタンと同じ動作だが、親指が届きやすい下部に配置 |
| ここに移動（下部・右） | インプット（操作トリガー） | なし | タップで現在表示中のフォルダを移動先として確定し、移動を実行してこの画面を閉じ、元の画面に戻る |

## 下部バーのレイアウト

```
┌──────────────────────┬──────────────────────┐
│      キャンセル      │      ここに移動      │
└──────────────────────┴──────────────────────┘
```

右上の×ボタンとキャンセルは同じ動作。下部に置くことで親指が届きやすくなる。

## 移動完了後の戻り先（確定事項）

「ここに移動」タップ後は、**移動前にいたフォルダの一覧に戻る**（移動先フォルダではなく元のフォルダ）。キャンセルの場合も同様に元のフォルダに戻る。

## 「戻る」と「キャンセル」の違い（重要）

- **画面内の右スワイプ／＜戻る**：このピッカー画面の中だけで1階層浅い場所に戻る。移動操作自体は継続中
- **キャンセル**：ピッカー画面ごと終了し、移動操作を中止して元の画面に戻る

## 移動先の整合性チェック（決定事項）

移動対象のフォルダは、**実の親フォルダの一覧に表示する際の1ヵ所だけ除外すれば十分**。子フォルダ・孫フォルダは「親フォルダをタップして初めて表示される」ため、入口となる移動対象フォルダ自身を隠せば、その配下すべてに自動的に到達できなくなる。子フォルダかどうかを再帰的にチェックする処理は不要。

## 未確定事項

- ピッカー画面の初期表示位置（常にルートフォルダから始めるか、移動対象が今いる場所から始めるか）は未確定
